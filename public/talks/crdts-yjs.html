Ideal state management

const state = { count: 0 }
state.count++

  1. Effortless reactivity
  2. Automatic offline support
  3. Always see latest - across all tabs
  4. Effortless sync with server

"CRDT" is just a definition, but available implementations are v. good

  • Conflict-free replicated data type on Wikipedia
  • Just any data structure where multiple processes can edit it, it can be replicated, and everyone eventually agrees on the state of the dang thing
  • Higher-level abstractions on top of CRDTs are getting good!
    • yjs
    • SyncedStore

Demo

  • davidharting/yjs-demo: A demo of a React app and a Node.js client both interacting with a yjs server
  • https://yjs-demo.fly.dev/

Caveats

  • This works if you are okay with every participant being equally a source of truth. The server is a peer!
  • In our context, this is
    • Not appropriate for representing the contents of a file
    • Is appropriate for representing the contents of an editor buffer
